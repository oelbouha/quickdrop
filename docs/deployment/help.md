# Help & Support

If you need support, use the following channels and templates. This repository also contains a `docs/` directory with legal documents (privacy policy) and operational notes.

## How to report a bug (recommended template)

When opening an issue or contacting support, provide:

- App version (from `pubspec.yaml` `version`)
- Platform (Android / iOS / Web / Desktop)
- Steps to reproduce
- Expected result
- Actual result
- Crash logs or screenshots (if available)
- Network logs (if the issue involves networking)

Create a GitHub issue or send an email to the project maintainer with the above information.

## How to request a feature

Open a feature request describing the user need, proposed behavior, and possible UI/UX sketches or steps.

## Contributing

Contributions are welcome. Typical workflow:

1. Fork the repository
2. Create a feature branch
3. Run `flutter pub get` and any linting/tests
4. Create a PR with a clear description and testing notes

Please follow the code style and keep changes focused.
